# Cross-Vendor Critic — Summary

_Aggregated from `eval/results/cross_vendor_*.json` (public-data runs
only). Generated by reading the JFK and Tier 3 ambiguous-Kennedy result
files; Moore-tree result files are gitignored and excluded._

## Why this experiment

Presentation feedback raised the concern that same-vendor models share
training data and possibly share failure modes — a Critic from the same
provider as the Hypothesizer might miss errors a model from a different
provider would catch. The cross-vendor experiment runs Anthropic Opus
4.7 and OpenAI GPT-5.5 on the same hypothesis set with shared seed state,
then compares verdicts.

Source: `eval/cross_vendor_critic_experiment.py` →
`eval/results/cross_vendor_<timestamp>.json`.

## Headline

On the unambiguous JFK case, both Critics agree (both accept). On the
ambiguous Tier 3 case (duplicate "John Kennedy b.1917 Boston" injected),
**Anthropic Opus 4.7 accepted both parental hypotheses while OpenAI GPT-5.5
flagged both as uncertain.** The verdict divergence on the ambiguous case
demonstrates that single-vendor evaluation can hide failure modes that
cross-vendor comparison surfaces.

## Per-case verdicts

### `cross_vendor_20260426_205511.json` — JFK (control)

Target: John F. Kennedy. Two parental hypotheses produced.

| Hypothesis | Anthropic Opus 4.7 | OpenAI GPT-5.5 | Disagreement? |
|---|---|---|---|
| `hyp:@I0@:@I1@:father` (Joseph Patrick Kennedy) | accept | accept | no |
| `hyp:@I0@:@I2@:mother` (Rose Elizabeth Fitzgerald) | accept | accept | no |

Both Critics accept both parental hypotheses. JFK's parentage is well-
documented public record; both vendors converge on the same verdict.

### `cross_vendor_20260426_210053.json` — Tier 3 ambiguous "John Kennedy b.1917"

Target: John Kennedy (the modified-Kennedy GEDCOM contains both JFK and
an injected duplicate persona with the same name and era). Two parental
hypotheses produced for the duplicate persona.

| Hypothesis | Anthropic Opus 4.7 | OpenAI GPT-5.5 | Disagreement? |
|---|---|---|---|
| `hyp:@I100@:@I101@:father` | accept | flag_uncertain | **yes** |
| `hyp:@I100@:@I102@:mother` | accept | flag_uncertain | **yes** |

Both hypotheses diverge: Anthropic accepts; OpenAI flags. OpenAI's
flag_uncertain matches the rubric for Tier 3 (genuinely ambiguous;
should produce `flag_uncertain` rather than confident accept). On this
case, OpenAI's behavior is the calibrated one.

## Asymmetric pairing experiment

`cross_vendor_hypo_*` files in `eval/results/` test the asymmetric
pairing: weak OpenAI Hypothesizer (gpt-4o-mini) with strong Anthropic
Critic (Opus 4.7). The recorded JFK and Tier 3 runs produced zero
hypotheses from the weak Hypothesizer in those particular invocations
— the experiment did not yield analyzable verdicts on the public-data
cases. The pairing pattern itself is implemented in
`eval/cross_vendor_hypothesizer_experiment.py` and re-runnable.

## Interpretation

The cross-vendor experiment supports two distinct claims:

1. **Vendor-independent confirmation strengthens unambiguous results.**
   When both vendors converge on accept (as on JFK's parentage), the
   verdict carries more weight than a single-vendor accept. A genealogist
   acting on the system's output can be more confident.

2. **Vendor disagreement on ambiguous cases is a useful signal.** The
   Tier 3 case shows OpenAI flagging while Anthropic accepts. Single-
   vendor evaluation would miss the ambiguity. The
   [multi-Critic ensemble](multi_critic_summary.md) builds on this
   observation and operationalizes it as a conservative aggregation
   rule.

## Reproducing this

```bash
python eval/cross_vendor_critic_experiment.py
```

Outputs land in `eval/results/cross_vendor_<timestamp>.json`, one file
per query. The JSON contains both Critics' full critique objects
(verdict, confidence, issues_found, justification) plus a
`comparison.per_hypothesis` block with side-by-side verdicts.
